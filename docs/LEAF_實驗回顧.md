## 五、模擬問答（5 個最棘手的問題）

### Q1：你的 95% 和 79.2% 是哪個？為什麼不一樣？

**簡答**：這是兩個不同的實驗。95% 是 20 題的 ReAct 策略消融（測 ReAct 循環 vs 單次調用），79.2% 是 50 題的四層品質保證消融（測 CoreVocab / RAG / Grammar / Verify 四個組件的貢獻）。測試集大小、題目內容、測的東西都不同。

**進一步解釋**：20 題版本排除了一些容易題（如母親/父親有髒數據），20 題全部是經過 corpus-verified 的基本詞彙。50 題版本包含更多類型（短句、長句、文化詞），所以基線更低但覆蓋更全面。

### Q2：你說 Multi-Agent 有價值，但你的 trace 顯示救回的 3 題只用了 KnowledgeAgent，Teaching 和 Quality 沒參與。Multi-Agent 的價值到底在哪？

**簡答**：在**翻譯消融實驗**中，優勢確實來自 ReAct 循環而非多 Agent 協作——這是我們的誠實結論。但在 **Agent 消融實驗**（15 題）中，單 Agent 翻譯類通過率 0%，4 Agent 100%——差異來自工具分工：Knowledge Agent 封裝了 RAG + Exact Match + Grammar + Verify，沒有它就沒有翻譯能力。

Multi-Agent 的核心價值是**架構可遷移性**：遷移新語言時只需更新 Knowledge Agent，Teaching 和 Quality 不用改。這是 7 分鐘可遷移的架構基礎。

### Q3：往返驗證（L4）在排灣語上是負效果（+4.1pp），這不是說你的框架有問題嗎？

**簡答**：L4 的負效果是**基座模型能力不足**造成的，不是框架設計問題。glm-4-flash 做反向翻譯（排灣語→中文）時品質太差，引入了額外噪聲。在更強的模型上，L4 預期為正向貢獻。我們的實驗誠實呈現了這個結果，並在建議中說明「用更強模型時才開啟 L4」。

### Q4：你一個人做，6 週，代碼量多少？可復現嗎？

**簡答**：框架代碼 4,560 行 Python（`agent_framework/`），加上 ASR、benchmark、UI 等，總計約 8,000-10,000 行。所有實驗數據都有 JSON 原始檔案，測試集、lexicon、ground truth 全部開源可追溯。語料來自 klokah 教材和族語E樂園（公開資源），FAISS 索引可本地重建。

### Q5：你的對手都在做 chatbot 或視頻生成，你做瀕危語言保護，應用價值在哪？
